# 시나리오 테스트 가이드

## 테스트 환경

- 도메인: `https://kadd.eregi.co.kr/2026spring`
- 브라우저: Chrome/Edge 최신 버전
- 시크릿 모드 권장 (캐시 영향 제거)

---

## 시나리오 1: 비회원 등록 시나리오

### 1.1 비회원이 ConferenceHome 접근

**테스트 절차**:
1. 시크릿 모드로 브라우저 열기
2. `https://kadd.eregi.co.kr/2026spring` 접근
3. 버튼 확인

**예상 동작**:
- ✅ "사전등록 신청하기 (Register)" 버튼 표시
- ✅ 아래에 "🔵 비회원으로 진행합니다..." 메시지 표시
- ✅ 버튼 클릭 시 `https://kadd.eregi.co.kr/2026spring/register?mode=guest`로 이동

**체크리스트**:
- [ ] 버튼이 "사전등록 신청하기"로 표시됨
- [ ] 비회원 안내 메시지가 보임
- [ ] 버튼 클릭 시 URL에 `?mode=guest`가 포함됨

---

### 1.2 비회원 RegistrationPage 진입

**테스트 절차**:
1. `https://kadd.eregi.co.kr/2026spring/register?mode=guest` 접근
2. Step 1 (기본 정보)로 이동
3. 비밀번호 UI 확인

**예상 동작**:
- ✅ "비회원 조회 비밀번호 (필수)" 라벨 표시
- ✅ 비밀번호 입력 필드 표시
- ✅ 비밀번호 안내 메시지 표시

**체크리스트**:
- [ ] 비밀번호 라벨이 표시됨
- [ ] 비밀번호 입력 필드가 보임
- [ ] 안내 메시지가 표시됨

---

### 1.3 비회원 등록 완료 후 페이지 나갔다가 재접근

**테스트 절차**:
1. 비회원으로 등록 진행 (결제 미완료 상태로 중단 가능)
2. 브라우저 닫기 또는 다른 페이지로 이동
3. 다시 `https://kadd.eregi.co.kr/2026spring` 접근

**예상 동작**:
- ✅ "사전등록 신청하기 (Register)" 버튼 표시
- ✅ "비회원등록조회 (Check Status)" 버튼 **안 보임**
- ✅ 새로운 등록이 가능해야 함

**체크리스트**:
- [ ] "사전등록 신청하기" 버튼이 보임
- [ ] "비회원등록조회" 버튼이 보이지 않음
- [ ] 새로운 등록이 가능함

---

### 1.4 비회원 결제 완료 후 재접근

**테스트 절차**:
1. 비회원으로 등록 완료 및 결제까지 완료
2. 브라우저 닫기
3. 다시 `https://kadd.eregi.co.kr/2026spring` 접근

**예상 동작**:
- ✅ "비회원등록조회 (Check Status)" 버튼 표시
- ✅ "이미 등록된 비회원입니다..." 메시지 표시
- ✅ 버튼 클릭 시 `https://kadd.eregi.co.kr/2026spring/check-status`로 이동

**체크리스트**:
- [ ] "비회원등록조회" 버튼이 보임
- [ ] 이미 등록된 비회원 메시지가 보임
- [ ] 버튼 클릭 시 등록 상태 확인 페이지로 이동

## 시나리오 2: 회원 등록 시나리오

### 2.1 회원이 로그인 후 ConferenceHome 접근

**테스트 절차**:
1. `https://kadd.eregi.co.kr/2026spring` 접근
2. "Log In" 버튼 클릭하여 로그인
3. ConferenceHome으로 다시 접근
4. 버튼 확인

**예상 동작**:
- ✅ "사전등록 신청하기 (Register)" 버튼 표시
- ✅ 버튼 클릭 시 `https://kadd.eregi.co.kr/2026spring/register?mode=member`로 이동
- ✅ 비회원 안내 메시지 **안 보임**

**체크리스트**:
- [ ] "사전등록 신청하기" 버튼이 보임
- [ ] 버튼 클릭 시 URL에 `?mode=member`가 포함됨
- [ ] 비회원 안내 메시지가 보이지 않음

---

### 2.2 회원 RegistrationPage 진입

**테스트 절차**:
1. `https://kadd.eregi.co.kr/2026spring/register?mode=member` 접근
2. Step 1 (기본 정보)로 이동
3. 비밀번호 UI 확인

**예상 동작**:
- ✅ 비밀번호 라벨 **표시 안됨**
- ✅ 비밀번호 입력 필드 **보이지 않음**

**체크리스트**:
- [ ] 비밀번호 라벨이 보이지 않음
- [ ] 비밀번호 입력 필드가 보이지 않음

---

### 2.3 회원 직접 URL 접근 (mode 파라미터)

**테스트 절차**:
1. 로그인 상태 유지
2. 브라우저 주소창에 `https://kadd.eregi.co.kr/2026spring/register?mode=member` 직접 입력
3. Enter

**예상 동작**:
- ✅ RegistrationPage로 정상 진입
- ✅ 회원 모드로 동작 (비밀번호 UI 숨김)

**체크리스트**:
- [ ] RegistrationPage가 열림
- [ ] 회원 모드로 동작함 (비밀번호 숨김)

---

## 시나리오 3: URL mode 파라미터 우선 순위

### 3.1 회원이 ?mode=guest로 접근

**테스트 절차**:
1. 로그인 상태 유지
2. `https://kadd.eregi.co.kr/2026spring/register?mode=guest` 직접 입력

**예상 동작**:
- ✅ 비회원 모드로 동작
- ✅ 비밀번호 UI 표시

**체크리스트**:
- [ ] 비회원 모드로 동작함
- [ ] 비밀번호 UI가 보임

---

### 3.2 비회원이 ?mode=member로 접근

**테스트 절차**:
1. 로그아웃 상태 (익명 세션)
2. `https://kadd.eregi.co.kr/2026spring/register?mode=member` 직접 입력

**예상 동작**:
- ✅ 회원 모드로 동작
- ✅ 비밀번호 UI 숨김

**체크리스트**:
- [ ] 회원 모드로 동작함
- [ ] 비밀번호 UI가 보이지 않음

---

### 3.3 mode 파라미터 없이 접근 (회원)

**테스트 절차**:
1. 로그인 상태 유지
2. `https://kadd.eregi.co.kr/2026spring/register` 접근 (mode 없음)

**예상 동작**:
- ✅ 로그인 상태로 fallback → 회원 모드
- ✅ 비밀번호 UI 숨김

**체크리스트**:
- [ ] 회원 모드로 동작함 (fallback)
- [ ] 비밀번호 UI가 보이지 않음

---

### 3.4 mode 파라미터 없이 접근 (비회원)

**테스트 절차**:
1. 시크릿 모드로 브라우저 열기
2. `https://kadd.eregi.co.kr/2026spring/register` 접근 (mode 없음)

**예상 동작**:
- ✅ 로그인 상태로 fallback → 비회원 모드
- ✅ 비밀번호 UI 표시

**체크리스트**:
- [ ] 비회원 모드로 동작함 (fallback)
- [ ] 비밀번호 UI가 보임

## 테스트 팁

1. **캐시 문제 방지**: 시크릿 모드 사용 또는 Ctrl+Shift+R 강력 새로고침
2. **로그인 상태 확인**: 브라우저 개발자 도구 → Application → Local Storage → Firebase Auth 확인
3. **URL 확인**: 항상 주소창의 `?mode=` 파라미터 확인
4. **비밀번호 UI 확인**: Step 1 (기본 정보)에서 "비회원 조회 비밀번호" 라벨 확인

---

## 버그 발견 시

1. 브라우저 개발자 도구 (F12) 열기
2. Console 탭에서 에러 메시지 확인
3. Network 탭에서 요청/응답 확인
4. 스크린샷 촬영 후 로그 기록
